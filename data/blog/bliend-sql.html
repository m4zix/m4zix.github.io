1. Detecting the Injection Point

  • Intercepted the request in Burp Suite.
  • Modified TrackingId to an invalid value ‚Üí noticed the message "Welcome back!" disappeared.
  • Confirmed application behavior depends on SQL query result ‚Üí potential blind SQLi.

2. Testing the Payload

Injected a condition to verify SQL execution:

' AND (SELECT 1 FROM users LIMIT 1)='1

Query executes successfully ‚Üí vulnerability confirmed.

3. Finding Password Length

Used a conditional payload to check password length of administrator:

' AND (SELECT 1 FROM users 
WHERE username='administrator' 
AND LENGTH(password) > 19)='1

Response confirmed password length = 20 characters.

4. Extracting Password Characters

Used SUBSTRING() to guess each character:

' AND (SELECT SUBSTRING(password,1,1) 
FROM users WHERE username='administrator')='a

Sent payloads to Intruder:

  • Payload 1: Position (1 ‚Üí 20)
  • Payload 2: Characters (a-z, 0-9, symbols)
  • Configured Grep - Match with keyword "Welcome" to detect correct guesses.

Slowly extracted the full administrator password.

üí° Lessons Learned

  • Blind SQLi relies on application behavior changes (not error messages).
  • LENGTH() helps determine size of sensitive fields.
  • SUBSTRING() allows extraction of data character by character.
  • Automation with Intruder speeds up brute-forcing the password.
